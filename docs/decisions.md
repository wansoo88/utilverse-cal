### ADR-001: Next.js 15 App Router + SSG 중심 아키텍처
- 날짜: 2026-03-16
- 상태: 채택
- 결정: Next.js 15 App Router를 사용하고, 모든 계산기 페이지를 SSG로 빌드
- 이유: SEO 최적화 필수 (검색 트래픽이 유일한 수익원), Vercel 무료 호스팅과 최적 호환, 정적 생성으로 로딩 속도 극대화
- 대안: Astro (더 가벼우나 인터랙티브 컴포넌트 많아 React 통합 복잡), Gatsby (SSG 특화지만 생태계 축소 중)
- 결과: 인터랙티브 계산기는 클라이언트 하이드레이션으로 처리, 맵은 dynamic import

### ADR-002: shadcn/ui 채택
- 날짜: 2026-03-16
- 상태: 채택
- 결정: 순수 Tailwind 대신 shadcn/ui 컴포넌트 라이브러리 사용
- 이유: 계산기 UI에 Select, Slider, Tabs, Toggle 등 인터랙티브 요소가 많음. Radix UI 기반 접근성 내장. 복사-붙여넣기 방식이라 번들 영향 최소. 다크모드 CSS 변수 전환 용이
- 대안: Headless UI (컴포넌트 수 적음), 직접 구현 (시간 소모, 접근성 누락 위험)
- 결과: 필요한 컴포넌트만 선택적으로 가져와 번들 최소화

### ADR-003: 정적 JSON 데이터 전략 (DB 없음)
- 날짜: 2026-03-16
- 상태: 채택
- 결정: DB 서버 없이 /data/*.json 파일로 모든 데이터 관리
- 이유: 월 $10 예산 제약, 데이터 변경 빈도 낮음 (월 1회), SSG 빌드 타임에 JSON import로 충분, 운영 복잡도 최소화
- 대안: SQLite (로컬 DB, 쿼리 가능하지만 SSG와 궁합 안 맞음), Supabase 무료 티어 (외부 의존성 추가)
- 결과: 데이터 업데이트 시 JSON 수정 → 재빌드 → 자동 배포. Phase 4 이후 API 자동 수집 검토

### ADR-004: Leaflet + OpenStreetMap + Open Charge Map (완전 무료 맵 스택)
- 날짜: 2026-03-16
- 상태: 채택
- 결정: Google Maps 대신 Leaflet.js + OSM 타일 + OCM API 조합
- 이유: 완전 무료 (Google Maps는 28,500 로드/월 초과 시 과금), 오픈소스, 글로벌 충전소 데이터 커버리지
- 대안: Google Maps (UX 우수하지만 비용 위험), Mapbox (50K 로드 무료지만 초과 시 과금)
- 결과: UX가 Google Maps보다 약간 떨어지지만 비용 0, 커스터마이징 자유도 높음

### ADR-005: 다크모드 기본 지원
- 날짜: 2026-03-16
- 상태: 채택
- 결정: 시스템 설정 감지 + 수동 토글 방식의 다크모드 지원
- 이유: 2026년 사용자 기대치, 계산기 장시간 사용 시 눈 피로 감소, 브랜드 차별화
- 대안: 라이트 모드 전용 (구현 단순하지만 사용자 경험 저하)
- 결과: CSS 변수 기반 테마 전환, next-themes 라이브러리 사용

### ADR-006: 글로벌 다지역 타겟팅 (미국 + 유럽 + 중국 + 오세아니아)
- 날짜: 2026-03-16
- 상태: 채택
- 결정: 미국 중심에서 유럽(독일, 노르웨이, 네덜란드, 영국), 중국, 호주/뉴질랜드까지 확장
- 이유: EV 보급률 상위 국가 커버리지 확대 → 트래픽 풀 확장, 유럽 EV 시장이 미국보다 성숙, 노르웨이는 신차 90%+ EV
- 대안: 미국 전용 (경쟁 치열, 트래픽 풀 제한)
- 결과: 지역 선택기에 미국 주 + 유럽 국가 + 중국 지역 포함, 통화/단위 자동 전환 ($/€/¥, miles/km), 전기요금 데이터 다지역 수집 필요
